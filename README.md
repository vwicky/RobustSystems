# RobustSmartSystems

Інженерний Python-проєкт для аналізу надійності ієрархічних резервованих систем (ІРС) з:
- GUI на `PyQt5` (`main.py`)
- CLI запуском (`main_cli.py`)
- математичним ядром у `src/formulas`
- інтерактивними графіками і структурними схемами

---

## Що робить проєкт

Проєкт моделює, як система деградує з часом, і відповідає на практичні питання:
- яка ймовірність, що система ще працює у момент `t`;
- коли очікується найбільша концентрація відмов;
- яка миттєва інтенсивність відмов;
- як резервування впливає на середній час життя системи.

---

## Фізичний зміст формул

### 1) Ймовірнісні характеристики

- `P_n(X,t)` — **ймовірність безвідмовної роботи**  
  Базовий показник надійності: шанс (0..1), що рівень `n` у конфігурації `X` працюватиме без поломки від `0` до часу `t`.

- `K_Γn(k,t)` — **коефіцієнт готовності (з резервуванням)**  
  Ймовірність, що підсистема ще працездатна в момент `t`, якщо для роботи достатньо щонайменше `k` справних елементів із `a`.

- `1 - K_Γn(k,t)` — **ймовірність відмови резервованої підсистеми**  
  Ймовірність, що до моменту `t` справних елементів стане менше, ніж `k`, і рівень вважається відмовленим.

### 2) Часові характеристики

- `T_n(X)` — **середній час безвідмовної роботи**  
  Очікувана тривалість життя базового стану (без урахування резервування). Формально — площа під кривою `P_n(X,t)` на `[0, +∞)`.

- `T_Γn(k)` — **середній час безвідмовної роботи з резервуванням**  
  Середня тривалість життя резервованого рівня (площа під `K_Γn(k,t)`), зазвичай більша за `T_n(X)`.

### 3) Характеристики невідновлюваних систем за заданої умови готовності

- `Q(k,t)` — **ймовірність відмови**  
  Накопичена ймовірність повної відмови до часу `t`. На старті `Q(0)=0`, з часом прямує до `1`.

- `a(k,t)` — **щільність відмов**  
  Похідна від `Q(t)`, показує концентрацію відмов у конкретний момент часу.

- `λ(k,t)` — **інтенсивність відмов**  
  `λ(t)=a(t)/(1-Q(t))` — миттєвий ризик відмови за умови, що система дожила до часу `t` без відмови.

### Підсумок логіки моделі

`P -> K_Γ -> T -> (Q, a, λ)`  
Від базової ймовірності безвідмовної роботи одного рівня — до комбінаторики резервування, середнього часу життя та швидкості/інтенсивності деградації всієї системи.

## Встановлення

### Вимоги
- Python `3.10+` (рекомендовано `3.11`)
- `pip`

### Кроки

1. Створіть віртуальне оточення:
   - macOS/Linux:
     - `python3 -m venv .venv`
     - `source .venv/bin/activate`
   - Windows (PowerShell):
     - `python -m venv .venv`
     - `.venv\Scripts\Activate.ps1`

2. Встановіть залежності:
   - `pip install -r requirements.txt`

3. Запуск:
   - GUI: `python main.py`
   - CLI: `python main_cli.py`

## Структура проєкту

- `main.py` — GUI-точка входу
- `main_cli.py` — CLI-точка входу
- `src/input_dataclass.py` — вхідні параметри варіантів + валідація
- `src/solve_calculator.py` — обчислювальне ядро
- `src/solver_module.py` — оркестрація задач і побудова графіків/схем
- `src/gui_module.py` — GUI та відображення результатів
- `src/formulas/` — математичні формули
- `compute_cache/` — кеш обчислень
- `tests/` — базові автоматизовані тести
