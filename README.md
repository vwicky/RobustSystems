# RobustSmartSystems

Інженерний Python-проєкт для аналізу надійності ієрархічних резервованих систем (ІРС).

Проєкт поєднує:
- GUI на `PyQt5` (`main.py`);
- CLI-режим (`main_cli.py`);
- математичне ядро у `src/formulas`;
- інтерактивні графіки та структурні схеми;
- експорт звітів `HTML + PDF` для презентацій/курсових.

---

## Що саме вирішує цей застосунок

Система моделює деградацію багаторівневої архітектури та дає практичні відповіді:
- яка ймовірність безвідмовної роботи у момент `t`;
- яка ймовірність відмови за умовою готовності (`k`);
- який середній час життя системи;
- коли очікується пік концентрації відмов;
- як змінюється миттєва інтенсивність відмов.

---

## Основні формули та зміст

### Ймовірнісні характеристики

- `P_n(X,t)` — **ймовірність безвідмовної роботи**: шанс, що рівень `n` у конфігурації `X` працює без відмови на інтервалі `[0, t]`.
- `K_Γn(k,t)` — **коефіцієнт готовності з резервуванням**: імовірність, що в момент `t` лишається щонайменше `k` справних елементів.
- `Q(k,t) = 1 - K_Γn(k,t)` — **ймовірність відмови** за умовою готовності.

### Часові характеристики

- `T_n(X)` — **середній час безвідмовної роботи** (площа під `P_n(X,t)` на `[0, +∞)`).
- `T_Γn(k)` — **середній час безвідмовної роботи з резервуванням** (площа під `K_Γn(k,t)`).

### Інтенсивнісні характеристики невідновлюваних систем

- `a(k,t)` — **щільність відмов** (похідна `Q(t)`).
- `λ(k,t)=a(t)/(1-Q(t))` — **інтенсивність відмов**: умовний миттєвий ризик відмови за умови, що система дожила до `t`.

### Логіка розрахунку

`P -> K_Γ -> T -> (Q, a, λ)`  
Від базових імовірностей рівнів — до оцінки готовності, ресурсу та швидкості деградації системи.

## Швидкий старт

### Вимоги
- Python `3.10+` (рекомендовано `3.11`);
- `pip`.

### Встановлення

1. Створіть та активуйте віртуальне середовище:
   - macOS/Linux:
     - `python3 -m venv .venv`
     - `source .venv/bin/activate`
   - Windows (PowerShell):
     - `python -m venv .venv`
     - `.venv\Scripts\Activate.ps1`

2. Встановіть залежності:
   - `pip install -r requirements.txt`

3. Запуск:
   - GUI: `python main.py`
   - CLI: `python main_cli.py`

## Експорт звітів (HTML + PDF)

Після обчислення в GUI доступна кнопка **"Експорт звіту (HTML + PDF)"**.  
Звіт містить:
- вхідні параметри варіанту;
- розраховані метрики та час виконання;
- формули (LaTeX-подання);
- текстові інтерпретації;
- графіки та структурні схеми.

---

## Структура проєкту

- `main.py` — GUI-точка входу;
- `main_cli.py` — CLI-точка входу;
- `src/input_dataclass.py` — вхідні дані та валідація;
- `src/solve_calculator.py` — обчислювальне ядро;
- `src/solver_module.py` — оркестрація задач, побудова графіків/схем;
- `src/gui_module.py` — GUI та відображення результатів;
- `src/report_exporter.py` — експорт звітів у `HTML`/`PDF`;
- `src/formulas/` — математичні формули;
- `compute_cache/` — кеш обчислень;
- `tests/` — автоматизовані тести.
